하망이의 무료 모바일 청첩장 만들기

GitHub Pages를 이용해서 무료로 나만의 모바일 청첩장을 만들어보세요. 개발 지식 없이도 할 수 있습니다.

완전 무료 · 코딩 필요 없음 · 30분이면 완성

하망이 부부가 결혼 준비를 하면서 모바일 청첩장을 직접 만들어 봤어요.
그 과정에서 알게 된 팁을 공유드리고자 합니다!

1

GitHub 회원가입

GitHub는 전 세계 수천만 개발자들이 사용하는 온라인 코드 저장소입니다. 코드를 저장하고 공유할 수 있으며, 웹페이지를 무료로 호스팅할 수도 있습니다.

  1. https://github.com/signup에 접속합니다.
  2. 이메일, 비밀번호, 사용자 이름(username)을 입력합니다.
  3. 이메일 인증을 완료하면 가입 끝!
사용자 이름(username)이 중요합니다!
나중에 청첩장 주소가 https://사용자이름.github.io/레포이름/ 형태가 됩니다.
예: 사용자이름이 honggildong이면 → https://honggildong.github.io/my-wedding/

3

내 정보로 수정하기 (config.js)

선택한 템플릿 폴더 안에 config.js 파일이 있습니다. 이 파일 하나만 수정하면 됩니다.

메모장, 텍스트 편집기 등 아무 프로그램으로 열어서 수정할 수 있습니다.

핵심: config.js의 내용 순서는 실제 청첩장에서 보이는 순서와 동일합니다. 위에서 아래로 읽으면서 수정하시면 됩니다.

수정 항목 체크리스트

  • 초대장 열기 화면 사용 여부 (useCurtain)
  • 신랑 이름 / 아버지 / 어머니
  • 신부 이름 / 아버지 / 어머니
  • 결혼 날짜, 시간
  • 예식장 이름, 층/홀, 주소, 전화번호
  • 카카오맵/네이버지도 링크
  • 인사말 문구
  • 우리의 이야기 문구
  • 신랑측 계좌번호 (신랑, 아버지, 어머니)
  • 신부측 계좌번호 (신부, 아버지, 어머니)
  • 링크 공유 시 나타나는 제목/설명

수정 예시

아래처럼 따옴표(" ") 안의 내용만 바꾸면 됩니다.

// 수정 전 (기본값)
groom: {
  name: "신랑",
  father: "아버지",
  mother: "어머니",
},

// 수정 후 (내 정보 입력)
groom: {
  name: "홍길동",
  father: "홍판서",
  mother: "김미영",
},
// 결혼 날짜/시간 수정
wedding: {
  date: "2026-05-09",      // 날짜 (YYYY-MM-DD 형식)
  time: "13:00",           // 시간 (24시간 형식)
  venue: "더채플앳청담",    // 예식장 이름
  hall: "그랜드홀 3층",     // 홀 이름
  address: "서울시 강남구 청담동 123",
  tel: "02-1234-5678",
},
주의사항
- 따옴표(" ")를 지우면 안 됩니다
- 쉼표(,)를 지우면 안 됩니다
- 줄바꿈을 넣고 싶으면 \n을 사용하세요
  예: "첫째 줄\n둘째 줄"
- 빈 줄(한 줄 띄기)을 넣고 싶으면 \n\n을 사용하세요
  예: "첫번째 문단\n\n두번째 문단" → 문단 사이에 여백이 생깁니다

초대장 열기 화면 설정

청첩장을 열었을 때 "초대장 열기" 버튼이 먼저 나오게 할지 설정합니다.

useCurtain: true,   // 초대장 열기 화면 보여주기
useCurtain: false,  // 바로 본문 보여주기

고인 표시 (故)

부모님이 돌아가신 경우 아래처럼 설정하면 자동으로 故 표시가 됩니다.

groom: {
  name: "홍길동",
  father: "홍판서",
  mother: "김미영",
  fatherDeceased: true,   // 아버지 고인 여부
  motherDeceased: false,  // 어머니 고인 여부
},

지도 링크 설정하기

  1. 카카오맵: map.kakao.com에서 예식장 검색 → "공유" 클릭 → URL 복사
  2. 네이버지도: map.naver.com에서 예식장 검색 → "공유" 클릭 → URL 복사
mapLinks: {
  kakao: "https://map.kakao.com/link/map/더채플앳청담,37.123,127.456",
  naver: "https://naver.me/abc123"
}

수정 후 확인하는 방법

config.js를 수정하고 저장한 뒤, 같은 폴더에 있는 index.html 파일을 브라우저에서 열어보세요. 이미 열려있다면 새로고침만 하면 됩니다.

GitHub에 올리기 전에 미리 확인하세요!
config.js 수정 → 저장 → index.html 새로고침 → 내용 확인 → 만족하면 GitHub에 업로드.
이 과정을 반복하면서 원하는 결과가 나올 때까지 수정하면 됩니다.

내 사진 준비하기

사진은 아래 폴더에 번호 이름으로 넣으면 자동으로 인식됩니다.

images/
  ├── hero/ ← 메인 사진 1장
  │   └── 1.jpg
  ├── story/ ← 스토리 사진들
  │   ├── 1.jpg
  │   └── 2.jpg
  ├── gallery/ ← 갤러리 사진들 (원하는 만큼)
  │   ├── 1.jpg
  │   ├── 2.jpg
  │   ├── 3.jpg
  │   └── ...
  ├── location/ ← 약도/지도 이미지 1장
  │   └── 1.jpg
  └── og/ ← 카카오톡 공유 썸네일 1장
      └── 1.jpg

사진 준비 규칙

폴더용도파일명권장 비율
hero/메인 대표 사진1.jpg3:4 세로
story/우리 이야기 사진1.jpg, 2.jpg ...자유
gallery/갤러리 사진들1.jpg, 2.jpg ...1:1 정사각형 권장
location/약도/지도1.jpg자유
og/카카오톡 공유 이미지1.jpg1200x630px 권장
파일명은 반드시 숫자.jpg로!
1.jpg, 2.jpg, 3.jpg ... 순서대로 넣어주세요.
중간에 빈 번호가 있으면 거기서 멈춥니다. (예: 1.jpg, 3.jpg만 있으면 1번만 표시)
사진 용량이 너무 클 때
사진이 크면 로딩이 느려집니다. iLoveIMG 같은 무료 사이트에서 압축하면 좋습니다. 사진당 500KB~1MB 이하를 추천합니다.

기존 샘플 사진 교체하기

템플릿에 미리 들어있는 샘플 사진을 내 사진으로 교체하면 됩니다.

  1. 각 폴더의 기존 샘플 사진을 삭제합니다.
  2. 내 사진 파일명을 1.jpg, 2.jpg ... 로 변경합니다.
  3. 해당 폴더에 넣습니다.

GitHub에 레포지토리 만들기

레포지토리(repository)는 파일을 저장하는 공간입니다. 하나 만들어야 합니다.

  1. GitHub에 로그인한 상태에서 오른쪽 위 + 버튼 → New repository 클릭
  2. Repository name에 원하는 이름 입력 (예: my-wedding)
  3. Public을 선택합니다. (무료 Pages는 Public만 가능)
  4. Create repository 클릭!

동영상으로 보기

레포지토리 이름 = 청첩장 주소의 일부
이름이 my-wedding이면 주소는 https://사용자이름.github.io/my-wedding/이 됩니다.
한글은 사용하지 마세요. 영문, 숫자, 하이픈(-)만 사용 가능합니다.

파일 올리기

수정한 템플릿 파일들을 GitHub에 올립니다. Git 설치 없이 웹에서 바로 올릴 수 있습니다.

  1. 방금 만든 레포지토리 페이지에서 uploading an existing file 링크를 클릭합니다.
  2. 선택한 템플릿 폴더를 열고, 폴더 안의 파일들만 전체 선택해서 드래그 앤 드롭합니다.
  3. 올릴 파일 목록을 확인합니다.
  4. 하단의 Commit changes 버튼을 클릭합니다.
템플릿 폴더 자체를 올리면 안 됩니다!
예를 들어 classic-elegant 폴더를 통째로 드래그하면 안 됩니다.
classic-elegant 폴더를 열어서 안에 있는 아래 파일들만 올려주세요:

  - config.js
  - index.html
  - script.js
  - styles.css
  - images 폴더

이 5개를 전체 선택(Ctrl+A / Cmd+A)해서 드래그하면 됩니다.
images 폴더는 통째로 드래그해도 됩니다!
images 폴더를 드래그하면 안의 하위 폴더(hero/, gallery/ 등)도 함께 올라갑니다.

동영상으로 보기

한 번에 100개 파일까지만 업로드 가능합니다.
파일이 100개가 넘으면 나눠서 여러 번 올리세요. (코드 파일 먼저 → 이미지 폴더 나중에)

수정이 필요할 때

배포 후에도 언제든 수정할 수 있습니다. 전부 GitHub 웹에서 가능합니다.

텍스트 수정 (이름, 날짜, 계좌번호 등)

  1. GitHub 레포지토리에서 config.js 파일을 클릭합니다.
  2. 오른쪽 위 연필 아이콘(Edit)을 클릭합니다.
  3. 내용을 수정합니다.
  4. 하단의 Commit changes를 클릭합니다.
  5. 약 1~2분 후 반영됩니다.

동영상으로 보기

사진 교체

  1. 교체할 사진의 파일명을 동일하게 맞춥니다. (예: 1.jpg)
  2. 해당 폴더에 Add file → Upload files로 새 파일을 올립니다.
  3. 같은 이름이면 자동으로 덮어쓰기 됩니다.

동영상으로 보기

사진 삭제

  1. 삭제할 파일을 클릭합니다.
  2. 오른쪽 위 휴지통 아이콘(또는 ... → Delete)을 클릭합니다.
  3. Commit changes를 클릭합니다.

전면 수정 (디자인 변경)

템플릿 자체를 바꾸고 싶다면, 가장 간단한 방법은:

  1. 기존 레포지토리를 삭제합니다. (Settings → 맨 아래 Delete this repository)
  2. 새 레포지토리를 만들고 다른 템플릿을 올립니다.
수정 후 바로 반영이 안 되면 브라우저에서 Ctrl + Shift + R (Mac: Cmd + Shift + R)로 새로고침 해보세요. 브라우저 캐시 때문에 이전 버전이 보일 수 있습니다.
카카오톡 공유 시 미리보기(제목, 설명, 썸네일) 수정은 config.js가 아니라 index.html을 직접 수정해야 합니다.
10번. 카카오톡 공유 시 썸네일·타이틀 수정하기를 확인해주세요.
10

카카오톡 공유 시 썸네일·타이틀 수정하기 (2026.4.28 업데이트)

카카오톡으로 링크를 공유하면 나타나는 제목, 설명, 이미지를 직접 설정할 수 있습니다.

GitHub에서 index.html 수정하기

  1. GitHub 레포지토리에서 index.html 파일을 클릭합니다.
  2. 오른쪽 위 연필 아이콘(Edit)을 클릭합니다.
  3. og:title이 있는 부분을 찾습니다. 아래처럼 수정해줍니다: 
    <meta property="og:title" content="타이틀을 넣어주시면 됩니다.">

자주 묻는 질문 (FAQ)

GitHub Pages 배포는 1~2분 정도 걸립니다. 잠시 기다린 후 다시 접속해보세요. 그래도 안 되면 Settings → Pages에서 Branch가 main으로 설정되어 있는지 확인해주세요.

아래 항목을 확인해주세요:

  • 파일명 확인: 반드시 1.jpg, 2.jpg 형태여야 합니다.
  • 대소문자 확인: 1.JPG와 1.jpg는 다릅니다. 소문자 .jpg로 맞춰주세요.
  • HEIC 파일 주의: 아이폰 사진은 HEIC 포맷입니다. 확장자만 .jpg로 바꾸면 안 됩니다! 반드시 JPG로 변환해야 합니다.
      - Mac: 미리보기에서 열기 → 파일 → 내보내기 → JPEG 선택
      - 웹: iLoveIMG에서 무료 변환
  • PNG 파일: PNG도 .jpg로 확장자만 바꾸면 안 됩니다. 위 도구로 JPG 변환 후 사용하세요.
  • 일부 사진만 안 보여요: 읽을 수 없는 파일은 자동으로 건너뜁니다. 해당 사진만 변환하면 됩니다.

브라우저 캐시 때문입니다. Ctrl + Shift + R (Mac: Cmd + Shift + R)로 강제 새로고침 해보세요. 그래도 안 되면 1~2분 더 기다려보세요.

- 따옴표(" ")가 빠진 경우
- 쉼표(,)가 빠진 경우
- 중괄호({ })가 빠진 경우

수정 전 원본 config.js와 비교해서 확인해보세요.

카카오톡은 한번 읽어간 URL의 정보를 캐시합니다. 카카오 공유 디버거에서 URL을 입력하고 캐시 초기화를 누르면 새 정보로 갱신됩니다.

카카오톡 공유 시에는 JavaScript를 실행하지 않고 정적으로 index.html만 읽기 때문에, config.js만으로는 불가능하며 직접 index.html을 수정해주어야 합니다. 이 점 제대로 확인하지 못한 점 죄송합니다.

사진은 파일명 번호 순서대로 표시됩니다. 순서를 바꾸고 싶으면 파일명 번호를 변경하면 됩니다. 예를 들어 3번째 사진을 1번으로 옮기고 싶다면, 파일명을 서로 바꿔주세요.

최대 50장까지 가능합니다. 다만 사진이 많으면 페이지 로딩이 느려질 수 있으므로, 10~20장 정도를 추천합니다. 사진 용량도 장당 500KB~1MB 이하로 압축하면 더 좋습니다.

네, 완전 무료입니다. GitHub Pages는 Public 레포지토리에 대해 무료로 웹 호스팅을 제공합니다. 도메인 비용도 없고, 트래픽 비용도 없습니다. 결혼식이 끝나면 레포지토리를 삭제하면 됩니다.

그냥 놔둬도 되고, 삭제하고 싶으면 레포지토리의 Settings → 맨 아래 Delete this repository를 클릭하면 됩니다.
